**Kmdo** 是 Kumo Search 生态专属的终端命令行工具（CLI），专为 Kumo 相关项目的全生命周期管理设计 —— 整合「开发调试、
版本发布、环境部署、配置管理」等核心能力，无需切换多套工具链，让开发者专注于业务逻辑，而非流程繁琐的环境配置与操作。

## 概述

Kmdo 是 Kumo 生态及后端服务（C++ / Go / Kumo Search）统一研发与上线的命令行工具。它提供统一的依赖管理、
批量构建与发布、多环境部署和操作可追溯等能力，让开发者脱离平台束缚，高效、标准化地管理项目全生命周期。

## 功能

Kmdo 围绕 Kumo Search 生态的核心需求，提供以下关键功能：

Kmdo 将 Kumo 生态及后端服务的开发、发布、部署、运维全流程整合为统一命令行操作，覆盖以下几大功能层面：

###  后端 C/C++ 开发依赖管理

Kmdo 可以在项目启动时自动生成统一依赖配置，管理所有第三方库的版本、功能选项、编译宏和构建参数，并自动解析依赖树，
确保每个模块使用的库版本和选项一致。团队成员在不同机器上拉取代码后，只需运行一次命令，就能得到完全相同的依
赖环境和构建结果，保证研发基线一致。

### 统一构建

在构建阶段，Kmdo 通过统一的 YAML 配置文件描述整个构建流程，包括依赖库版本、编译选项、构建参数以及二进制上传
地址等信息。构建完成后，产物自动上传至指定的二进制仓库，如 GitHub、GitLab、企业内部 Git 或 Kmup，实现全
团队构建产物的一致性和可追溯性。


### 集群部署

在部署阶段，Kmdo 根据构建阶段的产物进行统一分发和启动服务。部署过程严格基于研发基线，确保所有组件从开发到上线
使用的依赖和构建产物一致，实现全流程的可复现部署。

### 监控与日志

在服务启动阶段，Kmdo 根据配置自动集成 Prometheus 等常用监控工具，为各服务实例采集指标和日志，实现统一监控和可追溯的运行状态。

### 平台与集成

Kmdo 可作为独立工具执行依赖管理、构建、分发、部署及监控等全流程操作，无需依赖特定平台即可完成标准化开发与运维工作。同时，它也
能嵌入现有平台，通过平台调度执行任务，实现与平台的资源管理、服务启动、日志采集等功能的无缝协作。这种双模式设计既支持快速上手
的独立使用，也方便在大型团队或企业环境中统一集成，提高开发和运维效率。


### 生态联动能力

Kmdo 深度集成 Kumo Search 核心服务，通过命令行即可操作索引、执行数据同步和查询测试。它提供插件扩展机制，允许按需安装 Kumo 
生态的第三方工具，如性能监控或数据迁移插件。此外，Kmdo 与 Kumo Dashboard、Kumo SDK 等工具无缝对接，实现数据互通和跨工具
协作，支持复杂业务场景下的统一操作。

## 设计原则

### 脱离平台

Kmdo 不依赖任何平台，即可完成从依赖管理、构建、分发到部署和监控的完整流程。无论在本地环境、云端测试环境还是生产集群，都能独立运行，保证操作一致性和可复现性。

### 可嵌入平台

在需要的平台环境下，Kmdo 也能作为组件嵌入现有工具链，根据平台的调度完成相应职能，实现自动化管理和统一操作。

### 易用优先

所有操作通过统一命令行接口完成，遵循标准化流程，降低跨环境协作和团队成员上手成本。
