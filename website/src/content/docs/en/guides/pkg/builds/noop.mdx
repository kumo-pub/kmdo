You can now use `noop build` with Kmdo to create a pure data source package!

## Configuration

Simply set the `builder` to `noop`:


```yaml title=".kmdopkg.yaml"
builds:
  - builder: noop
```

For the data package section, see [Archives](/en/guides/pkg/archives/archives/).

## Complete Example

Refer to the kmcmake build configuration:

```yaml title="kmcmake/.kmdopkg.yaml"
version: 2

builds:
  - builder: noop
  
archives:
  - id: template
  - name_template: "{{ .ProjectName }}-{{ .Version }}-any-any"
    formats: [tar.gz]
    files:
      - src: "template"
        dst: template
      - src: "template/*"
      - src: "README.md"
      - src: "LICENSE"

checksum:
  name_template: "checksums.txt"

changelog:
  sort: asc
  use: github
  format: "{{ .SHA }}: {{ .Message }}{{ with .AuthorUsername }} (@{{ . }}){{ end }}"
  filters:
    exclude:
      - "^test:"
      - "^docs:"
      - "^ci:"
      - "merge"

release:
  github:
    owner: kumose
    name: kmcmake
  draft: false
  prerelease: false
  make_latest: true
  name_template: "v{{ .Version }}"
```
