kmdo is an automated release engineering tool developed in Go. It supports both `kminstaller` interactive 
installation and local index installation methods.  
It is compatible with Unix-like systems such as `Linux (amd64/arm64/arm-v7/i386)` and `macOS (amd64/arm64)`.
 The installation process requires no compilation and completes in seconds.

## I. Method 1: KM Installer Interactive Installation (Recommended, zero-configuration start-up)

Fully automated deployment is achieved through the official dedicated installation tool kminstaller, 
which automatically adapts to the system architecture, selects the optimal version, and configures
environment variables, making it suitable for most users to use quickly.

### Pre-dependency checks

kmdo installation only requires a Python 3.8+ environment (for running kminstaller). First, verify 
your local Python version:

```bash
# Output example: Python 3.10.12 (version ‚â• 3.8 is acceptable;

# If not installed, you need to execute: sudo apt install python3-pip -y first)
```


### 2. Install the kminstaller tool

```bash
pip install kminstaller --user
# Note: The --user parameter indicates user-level installation, which does not 
# require root privileges and avoids polluting the system environment.
```
:::tip [Solving the "kminstaller: command not found" problem]

If executing `kminstaller --help` results in a command not found error, it's because the Python 
user-level binary directory is not loaded. Execute the following command:

```bash source ~/.bashrc

```
Verification: If executing `kminstaller --help` displays help information, the configuration is successful.

:::

### 4. Start the kmdo interactive installation

```bash
  kminstaller install kmdo
```

The tool will detect the current system's OS and architecture, and automatically

The tool will perform the following operations without manual intervention:

## II. Method 2: Local Index Installation (Suitable for Offline/Internal Environments)

If your environment cannot access the external network (offline environment), or you need 
to use an internally customized version index, you can install via a local index file. The steps are as follows:

### 1. Prepare the local version index file

Obtain the kmdo_release_index.json (version index file) from official channels or the internal repository, and save
it to any local directory. Example:

:::note
- Example save path: ~/downloads/kmdo_release_index.json

- The index file contains information on all available versions (version number, release date, binary download link, etc.)
:::

### 2. Execute the local index installation command

```bash
kminstaller install kmdo --index ~/downloads/kmdo_release_index.json
# Note: The --index parameter is followed by the absolute or relative path to the local index file.
```

### 3. Installation Complete

- The tool will read the local index file and list all available versions.

- Subsequent steps are the same as interactive installation (select version ‚Üí automatic download/deployment ‚Üí 
  environment initialization)

- Environment variables will be automatically written to `~/.bashrc`, no manual configuration is required.

## IV. Installation Details

### 1. Default Installation Path (User-level Deployment, No Root Dependency)

- Core Program: `~/.kmdo/bin/kmdo` (Main executable file)

- Configuration Directory: `~/.kmdo/config/` (Stores default configuration and baseline data)

- Cache Directory: `~/.kmdo/cache/` (Stores dependency cache and temporary files)

- Installation Log: `~/.kmdo/installer.log` (Records installation process details for troubleshooting)

### 2. Environment Variable Configuration Instructions


`kminstaller` will automatically add the following configuration to `~/.bashrc`:

```bash
# kmdo environment variable configuration (auto-generated, do not modify manually)
export KMDO_HOME="/home/jeff/.kmdo"
# Source env file only if it exists (avoid startup errors)
[ -f "$KMDO_HOME/kmdo.env" ] && source "$KMDO_HOME/kmdo.env"
```

Function: Adds the `kmdo` installation directory to the system's `KMDO_HOME` for global access.

If using the `Zsh` terminal, you need to manually add the above configuration to `~/.zshrc` and then execute 
`source ~/.zshrc`.

`$KMDO_HOME/kmdo.env` exports variables managed by `kmdo` and command auto-completion.

`$KMDO_HOME/kmdo.env` automatically loads command prompts based on the different `shells`. The command prompt 
file is located in `$KMDO_HOME/completions`.

### 3. Supported Systems and Architectures

- Operating Systems: Linux(Ubuntu/Debian/CentOS, etc.), macOS(10.15+)

- Architecture Support: amd64(x86_64), arm64(aarch64), arm-v7(armv7l), i386(386)

## V. Uninstallation Methods

kmdo uses isolated deployment; simply deleting the installation directory will completely uninstall it, 
leaving no system remnants:

### 1. Basic Uninstallation (Delete Core Files)

```bash
rm -rf ~/.kmdo
```

Note: This command will delete all kmdo-related files (programs, configurations, caches, logs).

### 2. Clean Environment Variables (Optional)
Edit the `~/.bashrc` file and remove the automatically added PATH configuration line:

# 2. Find and delete the following lines

## VI. Troubleshooting Common Issues

### 1. Download Timeout/Failure

- Symptom: During installation, the message "Download failed (Request error): HTTPSConnectionPool read timed out" appears.
- Cause: Unstable network connection or restricted GitHub access.
- Solution:
  1. Check network connection and retry the installation command.
  2. Configure a proxy: `export HTTPS_PROXY=https://proxy_address:port`, then execute the installation.
  3. Switch to Method 2 (local index installation), manually download the binary package, and then deploy.

### 2. Decompression Failure

- Symptom: "Extraction failed: tarfile.ReadError: invalid header"
- Cause: Incomplete or corrupted binary package download
- Solution:
  1. Re-execute the installation command; the tool will automatically re-download.
  2. Manually download the corresponding version of the binary package and install using method 2 (--local parameter).

### 3. Initialization Failure

- Symptom: The message "kmdo init -f failed: ..." appears.
- Cause: Permission issues or configuration conflicts occurred during environment initialization.
- Solution:
  1. Manually execute the initialization command: `kmdo env init -f`
  2. Check the error log: `cat ~/.kmdo/installer.log` to locate the specific problem.

### 4. Version Index Loading Failed

- Symptom: "Local index file not found" or "Invalid index file format" message displayed.
- Cause: Incorrect local index file path or corrupted file.
- Solution:
  1. Check if the index file path is correct (absolute path takes precedence).
  2. Re-download the index file, ensuring the file format is valid JSON.

## VII. Advanced Use (Optional)

### 1. Custom Installation Path

To install kmdo to a custom directory (e.g., /opt/kmdo), you need to modify the kminstaller configuration 
(suitable for advanced users):

```text
# 1. Locate the kminstaller installation directory (execute `pip show kminstaller | grep Location`)
# 2. Edit the `kminstaller/component/kmdo/installer.py` file
# 3. Modify the default value of `install_dir`: `install_dir = "/opt/kmdo"` (root privileges required)
# 4. Execute the installation command: `sudo kminstaller install kmdo`
```

### 2. Offline Installation (No Network Environment)

1. Download from an external network environment:

- kminstaller installation package: `pip download kminstaller --no-deps -d ~/downloads/`
- kmdo version index file: `kmdo_release_index.json`
- Target version binary package: `kmdo-version-sys-arch.tar.gz` (Choose the architecture supported by your system based on sys and arch)

2. Copy to the offline environment and execute:
```bash

# Install kminstaller

pip install ~/downloads/kminstaller-xxx.whl --user

# Install kmdo (specify local package)

kminstaller install kmdo --index ~/downloads/kmdo_release_index.json --local ~/downloads/kmdo-xxx.tar.gz

```
