# *kmdo mirror tag 命令文档
## 概述
kmdo mirror tag 命令用于为镜像仓库中的二进制组件创建版本标签，并将关联的部署文件打包为压缩 tar 包，便于跟踪组件版本、简化分发流程，同时确保二进制文件与部署制品的一致性。该功能计划在 0.8+ 版本正式可用。

## 命令结构
kmdo mirror tag [TAG_NAME] [--options]

## 核心功能
1. 为镜像仓库中的二进制组件创建版本标签（TAG_NAME 为必填标识，用于版本区分）。
2. 将组件对应的部署文件打包为压缩 tar 包，便于分发和部署。
3. 关联组件版本与部署制品，保障二者一致性，简化版本追溯。

## 适用场景
1. 组件发布后创建版本标签，标记稳定可用版本。
2. 为特定环境（如生产、测试）创建专属标签，区分不同部署场景的制品。
3. 打包部署相关文件，与二进制组件一起分发，提升部署效率。

## 命令说明
### 用法
# 基础用法（指定版本标签）
```shell
kmdo mirror tag <TAG_NAME>

# 结合选项使用（具体选项待 0.8+ 版本补充）
kmdo mirror tag <TAG_NAME> [--options]

### 示例
# 为组件创建 v1.0.0 版本标签并打包部署文件
kmdo mirror tag v1.0.0

# 为测试环境创建专属标签（假设支持 --env 选项）
kmdo mirror tag test-v1.0.0 --env test
```
## 注意事项
1. 功能可用性：该命令计划在 kmdo 0.8+ 版本正式提供，低版本环境暂不支持。
2. 标签命名：TAG_NAME 建议遵循语义化版本规范（如 vX.Y.Z）或业务约定格式，便于识别版本信息。
3. 依赖配置：使用前需确保已配置有效的镜像仓库（可通过 `kmdo mirror show` 确认），且组件已发布到镜像仓库（可通过 `kmdo mirror list` 确认组件存在）。
4. 选项补充：--options 具体参数（如指定打包文件范围、标签描述等）将在 0.8+ 版本发布时补充，可参考在线文档获取最新信息。
5. 在线文档：详细使用说明可访问 https://docs.kumose.cc/kmdo/docs/cmd/mirror/tag 查看。
