# *kmdo mirror pull 命令文档
## 概述
kmdo mirror pull 命令用于从镜像仓库拉取二进制文件和部署文件，并打包为 tar 包，方便用户获取镜像中已发布的组件制品，支持版本化下载与本地部署。该功能计划在 kmdo 0.8+ 版本正式支持。

## 命令结构
kmdo mirror pull [ARGUMENTS] [--options]

## 核心功能
1. 从配置的镜像仓库中拉取指定组件的二进制文件。
2. 同步拉取该组件对应的部署相关文件。
3. 将拉取的文件自动打包为 tar 包，便于本地存储和使用。

## 适用场景
1. 从镜像仓库获取已发布的组件版本，用于本地部署或测试。
2. 同步组件的二进制文件与部署配置，确保部署环境一致性。
3. 批量拉取多个组件制品，简化多组件部署流程。

## 命令说明
### 用法
# 基础用法（具体参数待 0.8+ 版本补充）
kmdo mirror pull [组件名称] [版本/标签] [--options]

# 示例（基于功能推测）
# 拉取指定组件的最新版本制品
kmdo mirror pull mycomponent

# 拉取指定组件的特定版本制品
kmdo mirror pull mycomponent v1.0.0

# 拉取制品并指定输出 tar 包路径
kmdo mirror pull mycomponent --output ./downloads/mycomponent-v1.0.0.tar

## 注意事项
1. 功能可用性：该命令计划在 kmdo 0.8+ 版本正式提供，低版本环境暂不支持，升级后可使用。
2. 镜像配置：使用前需通过 `kmdo mirror show` 确认已配置有效的镜像仓库（本地工作区和远程源），否则无法拉取文件。
3. 组件依赖：拉取前需确保目标组件已在镜像仓库中发布（可通过 `kmdo mirror list` 确认组件存在）。
4. 参数补充：具体参数（如组件名称、版本/标签指定、输出路径、筛选条件等）及 --options 列表，将在 0.8+ 版本发布时补充。
5. 兼容性：拉取的 tar 包与 `kmdo mirror publish` 发布的制品格式兼容，确保上下游流程一致性。
