# kmdo mirror show 命令文档
## 概述
kmdo mirror show 命令用于查看当前生效的镜像配置，核心展示本地镜像工作区目录和关联的远程源URL，同时附加Git环境检测结果，帮助用户快速确认当前镜像配置状态及依赖环境可用性。

## 命令结构
kmdo mirror show

## 核心功能
1. 显示当前镜像的本地工作区目录路径。
2. 显示当前镜像关联的远程源URL。
3. 检测并显示Git环境状态（是否安装、安装路径及版本）。
4. 根据配置状态提供针对性操作提示（如未关联远程、未配置镜像时的引导）。

## 详细使用说明
### 用法
无需任何参数或Flags，直接执行命令即可查看配置：
kmdo mirror show

### 输出说明
根据当前配置状态和Git环境，输出分为以下几种场景：

#### 场景1：已配置完整镜像（本地目录+远程源）+ Git已安装
✅ Current Mirror Configuration
  Local Workspace: /Users/username/.kmdo/mirror
  Associated Remote: https://github.com/kumose-go/official-mirror.git
✅ Found git installed
  Location: /usr/bin/git
  Version: 2.39.3

#### 场景2：仅配置本地工作区（未关联远程）+ Git已安装
ℹ️  Hint: Local mirror workspace is set (/Users/username/.kmdo/mirror), but no associated remote.
  Use `kmdo mirror set /Users/username/.kmdo/mirror <remote-url>` to associate a remote source.
✅ Found git installed
  Location: /usr/bin/git
  Version: 2.39.3

#### 场景3：未配置任何镜像 + Git已安装
ℹ️  Hint: No valid mirror configured yet.
  Use `kmdo mirror set <local-dir> [remote-url]` to set one.
✅ Found git installed
  Location: /usr/bin/git
  Version: 2.39.3

#### 场景4：已配置完整镜像 + Git未安装
✅ Current Mirror Configuration
  Local Workspace: /Users/username/.kmdo/mirror
  Associated Remote: https://github.com/kumose-go/official-mirror.git
ℹ️ Git not installed: executable file not found in $PATH

## 输出字段说明
| 输出字段 | 含义 |
|----------|------|
| Local Workspace | 当前生效的本地镜像工作区绝对路径 |
| Associated Remote | 与本地工作区关联的远程Git仓库URL（未关联则不显示） |
| Found git installed | Git环境检测结果（已安装则显示路径和版本） |
| Location | Git可执行文件的安装路径 |
| Version | Git的版本号 |

## 注意事项
1. 命令无任何参数和Flags，执行后直接输出完整配置信息，无需额外配置。
2. 输出信息会根据实际配置状态提供针对性引导，帮助用户完成后续操作（如关联远程、配置镜像）。
3. Git环境检测结果用于提示镜像操作（如pull/push/clone）的依赖可用性，未安装Git时部分镜像功能可能无法使用。
4. 在线文档参考：https://docs.kumose.cc/kmdo/docs/cmd/mirror/show（可查看更详细的使用说明）。
