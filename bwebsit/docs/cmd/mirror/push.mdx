# *kmdo mirror push 命令文档
## 概述
kmdo mirror push 命令用于将本地镜像仓库中的二进制组件、关联部署文件及版本元数据，上传到已配置的远程镜像仓库。支持增量推送（仅传输新增或修改的制品）以优化网络效率，确保本地镜像的变更（如新增组件、更新版本）同步到远程，供多用户共享访问。该功能计划在 kmdo 0.8+ 版本正式支持。

## 命令结构
kmdo mirror push [--options]

## 核心功能
1. 上传本地镜像中的二进制组件到远程镜像仓库。
2. 同步组件关联的部署文件及版本元数据（如版本号、发布渠道、哈希值等）。
3. 支持增量推送机制，仅传输修改或新增的制品，减少网络带宽占用。
4. 同步本地镜像的 Git 提交记录到远程，确保本地与远程镜像状态一致。

## 适用场景
1. 本地发布组件后，将制品同步到远程镜像，供团队其他成员获取。
2. 本地镜像更新（如组件版本迭代、元数据修改）后，同步变更到远程。
3. 多环境共享镜像制品（如开发、测试、生产环境共用远程镜像）。

## 命令说明
### 用法
# 基础用法（默认同步所有变更）
kmdo mirror push

# 结合选项使用（具体选项待 0.8+ 版本补充）
kmdo mirror push [--options]

### 示例（基于功能推测）
# 增量推送本地所有变更到远程镜像
kmdo mirror push

# 强制推送（覆盖远程冲突，需谨慎使用）
kmdo mirror push --force

# 推送指定组件的变更（假设支持组件筛选）
kmdo mirror push --component mycomponent

## 前置条件
1. 功能可用性：需升级 kmdo 至 0.8+ 版本（低版本暂不支持）。
2. 镜像配置：已通过 `kmdo mirror set` 配置有效的远程镜像仓库地址（可通过 `kmdo mirror show` 确认）。
3. 网络连通：本地环境可访问远程镜像仓库（需确保网络通畅，及远程仓库认证信息已配置，如 SSH 密钥、用户名密码等）。
4. 本地变更：本地镜像仓库存在已提交的变更（如通过 `kmdo mirror publish` 发布的组件，或手动修改的元数据）。

## 注意事项
1. 增量推送机制：默认仅推送本地新增或修改的制品及元数据，未变更的内容不会重复传输，提升效率。
2. 冲突处理：若远程镜像存在与本地冲突的提交记录（如多人同时推送同一组件），需先通过 `kmdo mirror pull` 拉取远程最新变更并解决冲突后，再执行推送。
3. 认证要求：远程镜像仓库若需身份验证，需提前配置好认证信息（如 SSH 密钥配置、Git 用户名密码缓存等），否则可能推送失败。
4. 选项补充：--options 具体参数（如强制推送、组件筛选、日志级别等）将在 0.8+ 版本发布时补充，可参考在线文档获取最新信息。
5. 在线文档：详细使用说明可访问 https://docs.kumose.cc/kmdo/docs/cmd/mirror/push 查看。
