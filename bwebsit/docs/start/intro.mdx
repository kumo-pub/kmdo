# 简介

简单来说，kmdo 是一款集成开发依赖、自动化发布以、二进制管理、集群部署的工具。

目前支持 C++, Go、Rust、Zig、TypeScript（含 Bun 和 Deno）及 Python 语言。

C++语言支持从开发，集成，发布，部署全流程，并在基于基准线迭代，确保应用的相互之间
的运行稳定和接口兼容。

## 开发初衷

kmdo 脱胎于我们多年开发与运维的实战磨砺 —— 在一线场景中反复遭遇问题、拆解问题、沉淀解决方案，
最终将这些经实践验证的经验系统性收敛于一款工具。“易用” 与 “高效” 是贯穿设计全程的核心准则，
我们始终追求以最简洁的交互承载最强劲的功能，最终实现了 “工具形态、平台能力” 的产品落地：轻量
无依赖，可随时随地极速安装；全链路覆盖，能即刻满足从开发到部署的全流程需求。

# 全链路研发管理：基准一致·稳定复用的一站式解决方案
在长期开发运维实践中，开发依赖管理、发布流程管控、部署落地执行是贯穿全链路的三大核心痛点。多数团队最
终会通过编写脚本实现自动化，但这些脚本往往缺乏统一基准，导致复用性差、工具依赖繁杂、跨环境适配困难——
新机器部署或团队协作时需重复调试，既浪费时间，又难以保证流程一致性。

kmdo 以“统一基准线+稳定复用能力”为核心，将“工具形态承载平台能力”的理念落地，针对三大场景提供体系化
解决方案：无需零散脚本，仅通过简单 YAML 配置和单二进制文件，即可建立全链路统一基准，实现流程、配置、
能力的稳定复用，彻底摆脱重复劳动。

## 开发依赖管理：基准一致，复用无虞
### 核心问题
开发阶段的“基准混乱”是复用的最大障碍：不同开发者本地依赖版本、依赖源不一致，导致“我这能跑”的
协作困境；多语言项目依赖管理工具碎片化，缺乏统一配置基准，新成员上手需重新适配；依赖配置未
固化，团队协作或CI环境搭建时需重复调试，复用成本高。

### 设计思考
要实现依赖管理的稳定复用，核心是建立“统一基准线”——将依赖源、版本约束、私有仓库凭证等关键配置固化，让所
有环境（开发、测试、CI）都基于同一基准执行；同时适配多语言原生规范，不额外增加学习成本，让配置和依赖环
境可直接复用。

### 解决方案
kmdo 将依赖管理基准固化到标准化 YAML 配置中，自动适配 Go、Rust、Python 等语言的原生依赖规范，统一定义
依赖源、版本约束、缓存策略等核心信息，形成团队统一的依赖基准线。无论是新开发者加入、CI环境搭建，还是跨项目
复用依赖配置，只需直接复用该 YAML 基准，即可快速构建一致的依赖环境，无需重复调试。同时内置依赖全景分析
与校验能力，确保基准配置的合规性，从根源上消除“环境差异”，让依赖管理的复用稳定、高效、无风险。

## 发布管理：流程固化，一键复用
### 核心问题
发布流程缺乏统一基准，导致复用困难：不同项目的发布脚本风格各异、步骤零散，跨项目复用需大幅修改；人工操作或脚本
碎片化导致发布步骤不统一，易出现版本标签错误、签名遗漏等问题；发布配置未固化，新发布任务需重复编写流程，效率
低下且风险不可控。

### 设计思考
核心是“流程基准化、配置可复用”——将发布全流程的关键步骤（构建、打包、签名、归档）固化为统一基准，让“配
置即基准流程”，一次配置即可在多个项目、多次发布中稳定复用；通过工具自动化执行基准流程，避免人工干预导致
的基准偏移，确保每一次发布都一致可控。

### 解决方案
kmdo 支持在 `.kmdo.yaml` 中固化发布全流程基准，包括构建参数、打包格式、签名规则、校验和生成、归
档目标等核心配置，形成统一的发布基准线。该基准配置可直接跨项目复用，只需根据项目特性微调参数即可。工
具会严格遵循基准流程，自动基于 Git 标签确认版本号，完成多平台制品构建、依赖合规校验、签名与 SBOM 生
成、归档发布等全环节操作。无论是同一项目的迭代发布，还是跨项目的标准化发布，都能一键复用基准流程，既保
证了发布质量的一致性，又彻底摆脱重复编写脚本的繁琐，实现高效复用。

## 部署管理：环境适配，基准复用
### 核心问题
部署环节的“基准割裂”导致复用成本高：生产、测试、预发等多环境缺乏统一部署基准，配置零散且易冲突；不
同部署目标（物理机、容器、边缘节点）需编写不同部署脚本，无法复用；部署流程未固化，新环境部署或跨项
目部署时需重复调试适配，且难以保证部署效果一致。

### 设计思考
要实现部署的稳定复用，需建立“环境无关的基准流程+可适配的环境配置”——将部署的核心流程（制品分发、配
置渲染、服务启停、校验回滚）固化为统一基准，同时将环境差异化配置分离，让基准流程可跨环境复用；工具
本身需打破环境依赖，确保基准流程在不同场景下都能稳定执行。

### 解决方案
kmdo 以单二进制文件形态存在，无额外依赖，可在 Linux、macOS 等多系统多架构环境中稳定运行，确保基
准流程的跨环境兼容性。支持在 YAML 中分离定义“基准部署流程”与“多环境差异化配置”：基准流程固化制品分
发、服务启停、校验回滚等核心步骤，多环境配置则单独定义目标地址、部署路径、依赖预加载清单等。跨环境或
跨项目部署时，可直接复用基准流程，仅需切换环境配置即可，无需修改核心流程。部署时工具会严格遵循基准流
程执行，自动适配目标环境，提前校验依赖兼容性，确保部署效果一致；搭配一键回滚机制，实现基准稳定、配置灵活
、复用高效的部署管理。
## 它好用吗？

在过去几年中，kmdo 已被 Go 社区广泛采用，已有数千个项目和公司使用它来管理发布流
程。


## 适用场景

kmdo 专为 CI 工具设计——你只需在构建脚本中下载并执行它即可。

是否在本地机器安装完全取决于你，当然这也是可行的。

## 使用方法

整个发布流程通过 `.kmdo.yaml` 配置文件进行定制。

配置完成后，每次需要创建新版本时，只需推送一个 Git 标签，然后运行 `kmdo release` 命令,

你也可以在任意主流持续集成平台中使用它。
